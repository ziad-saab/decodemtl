DecodeMTL On-Demand

Welcome to DecodeMTL :)

If you are reading this document, it's either because you signed up to one of our upcoming full-time web development bootcamps, or because you signed up to our on-demand online web development course.

Either way, this preparatory material is the first step in your journey to become a web developer. The material is designed to get your feet wet with web development, and help you answer the question of whether or not you want to pursue your web development learning further.

Objectives

By going through this preparatory material, you should gain a good general understanding of web development. You will learn the following things:

  • The very fundamentals of HTML and CSS
  • Some cool tricks to make you a CSS-ninja
  • The very fundamentals of JavaScript
  • How to use Git and GitHub to manage code and collaborate with a team
  • The very fundamentals of your computer's command line interface

Commitment

This preparatory material will take anywhere between 30-40 hours to complete. If you are applying to our full-time bootcamp, completing this material before starting the bootcamp is mandatory. Failing to do so, you will be playing a catch-up game during class.

Community and Mentoring

You are not expected to complete all this material by yourself. The rule of thumb is: If you get stuck for more than 15 minutes, be sure toget in touch with us for hints and/or explanations.

Once you sign-up, you will be invited to our Slack channel where you can communicate with peers who will be at various stages of completion of this material. The channel is also staffed with one or more mentors who can help answer your questions about the material.

Whether you signed up to our full-time bootcamp or taking this course as a standalone introduction to web development, you will have received a separate instructions for 1-on-1 mentoring. We strongly encourage you to make use of this private mentoring time to solidify your understanding of the subjects we teach.

Assignments

Throughout the course, you will be required to complete some assignments.

Getting Started!

Before getting started, we'll need to equip ourselves with some tools.

  • Download Google Chrome

    Google Chrome will be used as the main development browser during the training. Its integrated set of debugging tools makes developers' lives easier. Chrome's rendering engine is used by multiple browsers, giving you a good idea of what your pages would look like in many different settings.

  • Signup for a GitHub account

    Git and GitHub will be a constant part of our workflow. While Git will enable us to keep track of the history of code that we write and manage variations in our code, GitHub will enable us to share and collaborate the code we write.

    For this section, you are only tasked with signing up for a GitHub account. The last section of the prep material deals with your introduction to Git and GitHub.

    This is a good place to mention that your GitHub username and profile have the same visibility as your profiles on facebook, twitter or LinkedIn. GitHub is the network of code in the same manner that LinkedIn is the network of professionals. Employers will look at your GitHub profile for signs of your experience and interest. At this point, make sure to create a profile that represents you. Later on, we will fill your GitHub with various projects that will demonstrate the knowledge you have learned through DecodeMTL.

  • Signup for Cloud9 IDE

    Just like we edit our spreadsheets in Open Office or Excel, web pages and web applications need their own editor. Cloud9 is an IDE — Integrated Development Environment — in the cloud. This is just technical mumbo-jumbo. It's a text editor on steroids, and already has installed most of the tools we will need during the bootcamp. Everything you do in Cloud9 is saved online, so you are able to start coding from any computer that has internet access. You don't even need to install anything on your machine.

    Make sure to use the "Sign in with GitHub" option when signing up to Cloud 9. This will link your GitHub account to your Cloud 9 account, allowing for an easier workflow between your code editor and source code repository.

HTML & CSS

HTML and CSS are the fundamental languages of the web. Using only these two languages, we are able to create compelling, informative landing pages and micro-sites.

For this part of the preparatory course, you will be going through some online classes covering the bases of HTML and CSS.

  • Codecademy's intro to HTML & CSS

    Complete all the tracks of this course.

  • Code School's CSS cross coutry

    Complete only level 1 of this course

  • Code School's Chrome Dev Tools

    Go through levels 1 and 2 only. This will give you a good understanding of the tools we use on a daily basis to help us find bugs in our web pages.

  • Assignment: Create a page about your favourite movie

    The goal of this HTML and CSS assignment is to review the material of this section. The instructions below should be viewed as a starting point only. You are encouraged to experiment as much as you like on the technologies that you just discovered.

    Using the Cloud 9 IDE, create a new public project. Inside this project, create a file called movie.html. Create the content for your movie as you see fit. Try to use as much semantic HTML as possible. At the very minimum, your page should have the movie's title, a poster, description, categories/tags and trailer. Perhaps you could even add a photo grid of action shots from the movie? Use your imagination and have fun :)

    Make sure to start only with your semantic HTML and add the CSS later on. Your page will look boring at first, but you'll quickly be able to change that using some CSS. Working on the HTML and CSS separately has some advantages. It lets you concentrate on one thing at a time without distractions. Separating concerns also shows you that you can easily change the style of your page without touching the content at all. A great example of this way of doing things is the CSS Zen Garden: this showcase of CSS code is all based on the exact same HTML markup. If you are curious, you can even take a look inside using your Chrome Development Tools!

    Once you are satisfied with your HTML content — you should have at least <section>, <h1>, <ul>, <li> and <p> tags — you should start working on the CSS for the page. Here we will only give you some directives, and let you use your imagination for the rest.

    Try to make it so that the poster and main content are flowing next to each other using floats. Find a place or two that could benefit from a :hover effect. Try to use CSS transitions to make the hover effect smooth. If it's appropriate to your choice of design, use rounded corners on the movie's cover or on the action shots. Perhaps use a custom font through Google Fonts? Again here, give free reign to your imagination and creativity. Remember to look at Zen Garden for inspiration.

    Throughout this process, remember to refresh the page as often as possible in Cloud 9's preview window. This iterative process is how your fellow web developers work on a daily basis. Don't forget to use Chrome's Developer Tools to help you tweak your CSS fast, then copy the result in your text editor.

Javascript & jQuery

JavaScript originated as a language to add dynamic functionality to a web page. Over the years, it has evolved into a platform for creating full web-based applications on the front-end, as well as a back-end platform mainly through NodeJS.

Even though this part of the preparatory material deals with JavaScript from a front-end perspective, a lot of the general programming concepts will apply on the back-end as well when we look at NodeJS. Since JavaScript is an integral part of this bootcamp, this section is the most important.

  • Codecademy's JavaScript track

    Go through all the chapters of this track.

  • Codecademy's jQuery track

    Go through all the chapters of this track.

  • Assignment: Adding functionality through jQuery

    This assignment picks up where we left off with the HTML and CSS assignment. Here, we will add some dynamic functionality to the movie page we created. Accomplishing this assignment will require a great deal of resourcefulness and patience. You might need our help too, so don't forget to stay on the Slack channel at all times.

    For this assignment, the goal is to enable the user viewing the page to suggest new tags/categories that we should add to the movie listing. Transmitting this suggested data to us will require a back-end of some sort to receive, store and manage the data but this is not part of the assignment. Here, we will only concentrate on the front-end part of the assignment: giving the user an interface through which they can add new tags and / or categories, and a button to send them. In this version of the assignment, when "sending" the suggested tags we will simply alert them back to the user.

    The requirements of the assignment are purposely vague. For one, this mimicks a real company environment where work often has to start before requirements are fully set in stone. This will also give you the opportunity to communicate with us to clarify the requirements.

NodeJS: JavaScript for the back-end

Now that you have completed the section on JavaScript and jQuery, you should be somewhat introduced to the JavaScript language. Even though we will be using the same language on the back-end, the concepts are not all the same: front-end JavaScript deals more with UI logic, and less with data and I/O (input/output). On the back-end side, we will be using JavaScript to talk to databases and other back-ends (APIs).

In JavaScript, all these I/O operations happen asynchronously, almost like sending a message in a bottle. This section introduces you to some of the concepts that will come up when doing JavaScript on the back-end.

Git & GitHub

History management is an integral part of writing software. Throughout the history of a piece of software, be it a web site, web application or other, changes will happen. Often, features that are not ready for prime-time will have to be developed in parallel. And in most cases, software will be worked on by multiple developers.

Git is a tool that empowers developers to accomplish these features without having to worry about the state of the project. If you've ever written a Word document, you may have found yourself creating a copy at some point, in order to try something else. Among other things, Git gives you the ability to branch off and create new features while keeping the working version of your software clean. It also lets you go back in time to a previous working version and examine it, without ever losing your work.

GitHub is like a "social network around Git". It is a web site that lets you publish your Git-managed projects online for others to see and collaborate on. Your GitHub profile is the equivalent of your LinkedIn profile for technical employers. In it, anyone can see what projects you have shared and contributed to. This is why it is important to document your adventures in web development by publishing as much of your work as possible to GitHub.

For this part of the course, we will start by learning to use Git. Git is a program that you have to install on your system. Fortunately in our case, since we are doing all our work inside the Cloud 9 IDE, Git is already installed for us. Cloud 9 has even already configured our Git installation to tag all the work we do with our name and e-mail address.
